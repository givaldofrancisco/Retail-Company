src/graph/ — Workflow Orchestration

The core LangGraph state machine. Defines the workflow nodes, routing logic, and shared state that flows through the entire pipeline.

LangGraph State Machine
◈ workflow.py CORE Builds the LangGraph graph with 12 nodes — classify, schema, golden retrieval, SQL gen/validate/execute/repair, sanitize, report, finalize — with conditional routing
◈ nodes.py CORE Dataclass containing all node implementations — each method takes AgentState and returns updated state
◈ state.py AgentState TypedDict — defines all fields (request metadata, SQL candidates, query results, errors, final report)

src/llm/ — LLM Integration

Multi-provider LLM client with fallback strategies. Supports both Gemini and OpenAI, with heuristic SQL generation when all providers fail.

Language Model Client
◈ client.py CORE Multi-provider LLM wrapper — safe_invoke with error handling, heuristic SQL fallbacks, methods for SQL generation, repair, and report generation
◈ prompts.py Prompt builders — assembles question, schema context, golden examples, and format instructions into structured prompts for each LLM task

src/tools/ — External Tool Integrations

Connectors to BigQuery, the Golden Knowledge Bucket, schema introspection, and SQL validation. The tools that the LangGraph nodes call.

Data & Query Tools
◈ bigquery_runner.py DATA BigQuery client — executes SQL queries against bigquery-public-data.thelook_ecommerce and fetches table schemas
◈ golden_retriever.py Loads golden Q/SQL/Report trios from JSON, retrieves top-k similar examples using token-based scoring for in-context learning
◈ schema_tool.py Schema introspection for allowed tables (orders, order_items, products, users) with hardcoded fallback on BigQuery API failure
◈ sql_validator.py SAFETY Enforces SELECT-only, blocks DML/DDL, restricts table references to allowlist, validates dataset, appends LIMIT clause

src/safety/ — Guardrails & PII Protection

Intent classification and PII masking layer. Ensures the agent never exposes sensitive data and handles destructive requests safely.

Safety Layer
◈ guardrails.py SAFETY Intent classifier — routes user input to analysis / schema_lookup / unsupported / destructive_report_op using keyword and regex patterns
◈ pii_masker.py SAFETY Removes columns with PII names (email, phone) and masks residual PII patterns in text using regex substitution

tests/ — Test Suite

27 automated tests across 7 suites. Covers safety, SQL validation, repair loops, guardrails, acceptance, LangGraph memory, and LLM resilience.

◈ test_pii_masking.py SAFETY Validates PII column removal and email/phone regex masking across DataFrames and text
