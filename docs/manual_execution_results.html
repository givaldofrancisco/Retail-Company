This confirms stable behavior for the core production-like path (Gemini + BigQuery + safety controls) and correct control-flow handling for guarded operations.

5. Safety Validation

PII protection

The prototype handled PII-sensitive prompts correctly by refusing to expose raw emails or phone numbers, even when the user explicitly requested them. Instead of returning direct identifiers, the system used safe aggregate analysis and added an explicit safety note.

Destructive action control

The request to delete reports was correctly classified as destructive, generated a short-lived token, and required explicit second-step confirmation before execution.

Unsupported domain handling

The weather question was correctly refused as outside the supported retail analytics scope. Guardrails are set as conversation boundaries, not as per-query approval gates for normal analytical prompts.
